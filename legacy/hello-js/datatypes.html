  • zzNumbers

    Числовой тип данных включает в себя целочисленные значения и числа с плавающей точкой, а также специальные числовые значения (Infinity, -Infinity, NaN)
    NaN означает ошибку вычисления
    Любое действие с NaN возвращает NaN

    Математические операции безопасны, т.е. скрипт никогда не остановится с фатальной ошибкой, в худшем случае будет получено значение NaN

  • BigInt

    В JS тип number может лежать в пределах от -253 до 253. Тип BigInt позволяет работать с числами произвольной длины
    Чтобы создать переменную типа BigInt, необходимо добавить символ 'n' в конце числа
    let bigNumber = 12345678901234567890123456789999n;

  • String

    Строка должна быть заключена в кавычки
    let str1 = "Строка раз"; //двойные кавычки
    let str2 = 'Строка два'; //одинарные кавычки
    let str3 = `Строка три`; //обратные кавычки (апострофы)
    Одинарные и двойные кавычки равносильны в JS
    Обратные кавычки позволяют встраивать в строку выражения, значение которых вычисляется и становится частью строки
    let name = "Бонифаций"
    alert(`Ну здравствуй, ${name}.`); //Ну здравствуй, Бонифаций.
    alert(`Посчиталось: ${19+3}.`); //Посчиталось: 22.

  • Boleean

    Булевый тип данных имеет только два значения, true - да, истина, верно и false - ложь, неверно
    Результат сравнения тоже имеют булевый тип данных

  • null

    Специальное значение null не относится ни к одному типу данных
    Данное значение (а оно же и тип данных) обозначает "ничего" или "значение неизвестно"

  • undefined

    Значение undefined обозначает, что переменной не было присвоено никакое значение
    let x;
    alert(x); //undefined
    Технически undefined можно присвоить переменной, но делать это не принято, так как для неизвестного значения служит null, а undefined используется в качестве проверки, было ли присвоено переменное что-либо

  • Object

    Объекты служат для хранения коллекций данных или других объектов
    Тип symbol служит для создания уникальных идентификаторов объектов

  • Оператор typeof

    Оператор typeof возвращает тип аргумента
    Есть два синтаксиса данного оператора, оба правильные
    typeof(x);
    typeof x;
    Вызов typeof возвращает строку с названием типа
    typeof undefined; //"undefined"
    typeof 0; //"number"
    typeof 10n; //"bigint"
    typeof true; //"boolean"
    typeof "foo"; //"string"
    typeof Symbol("id"); //"symbol"
    typeof Math; //"object"
    typeof null; //"object"
    typeof alert; //"function"
    В данном примере typeof null; действительно вернет значение object, однако это не так, это официально признанная ошибка в typeof
    Типа данных function не существует, однако для удобства typeof возвращает именно это значение
