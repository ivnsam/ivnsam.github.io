  • Цикл "while"

    Еще этот цикл называют циклом с предусловием
    Синтаксис
    while (continue) {
    //тело цикла
    }
    Тело цикла будет выполняться до тех пор, пока условие continue истинно
    Условие цикла приводится к логическому виду

  • Цикл "do...while"

    Еще этот цикл называют циклом с постусловием
    Синтаксис
    do {
    //тело цикла
    } while (continue)
    В данном цикле сперва выполнится тело цикла, а затем проверится условие
    Такой цикл выполнится как минимум один раз

  • Цикл "for"

    Синтаксис
    for (начальные данные; условие завершения; шаг) {
    //тело цикла
    }
    В данном цикле в самом начале один раз задаются начальные данные, затем проверяется условие завершения цикла, если оно не false, то выполняется тело цикла, затем выполняется то, что задано в блоке шаг. И так будет происходить пока условие завершения не станет равным false

    Встроенное объявление переменной

    Переменную счетчика можно объявить в самом цикле, в начальных данных. В таком случае данная переменная будет недоступна в других местах кода
    Если переменную объявить вне цикла, то она будет доступна вне цикла
    let i = 0; //переменная объявлена вне цикла
    for (i=0; i<3; i++) alert("cycle 1");
    alert(i); //поэтому переменная доступна вне цикла
    for (let j=0; j<5; j++) alert("cycle 2"); //переменная объявлена в цикле
    alert(j); //поэтому перменная не доступна вне цикла

    Пропуск частей цикла for

    Любая часть инициализации цикла может быть пропущена при необходимости
    for (; i<3; i++) alert("1. пропущена инициализация начальных условий цикла");
    for (let i=0;; i++) alert("2. пропущено условие окончания цикла");
    for (let i=0; i<3;) alert("3. пропущен шаг цикла");
    В первом случае пропущена инициализацию начальных условий цикла, если для них нет необходимости (например используется уже созданная переменная)
    Во втором случае пропущено условие окончания цикла. В таком случае цикл будет длиться бесконечно
    В третьем случае пропущен шаг цикла. Его можно пропустить, например, если в теле цикла есть изменение переменной шага цикла
    Можно комбинировать все эти варианты как угодно. Можно даже пропустить все части инициализации цикла, получив при этом вечный цикл, однако точки с запятой обязательно должны присутствовать, иначе будет ошибка синтакиса
    for (;;) {//здесь получится вечный цикл}

  • Директивы break и continue

    break

    Директива break служит для завершения выполнения цикла и выхода из него
    let sum = 0;
    while (1) {
    let number = +prompt("Введите число", "");
    if (!number) break; //если введено не число или ничего не введено, выведется сумма через alert, тоесть произойдет выход из цикла
    sum += number;
    }
    alert(`Сумма равна ${sum}`);

    continue

    Директива continue служит для перехода к следующей итерации цикла
    for (let i=0; i<15; i++){
    if (i%2 == 0) continue; //если число четное, то выведено не будет, так как произойдет переход к следующей итерации
    alert(i);
    }
